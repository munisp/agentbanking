KEY FEATURES
════════════

✅ Real-time notifications (< 100ms latency)
✅ Automatic reconnection with exponential backoff
✅ Sound notifications (beep on transaction)
✅ Haptic feedback (vibration)
✅ Persistent WebSocket connections
✅ Notification history with read/unread status
✅ Connection status indicator
✅ Works with mobile networks (3G/4G/5G)
✅ Battery efficient (WebSocket + ping every 30s)
✅ Supports thousands of concurrent agents


TECHNOLOGY STACK
════════════════

Backend:
  - Python 3.10+
  - FastAPI (WebSocket support)
  - SQLAlchemy (PostgreSQL)
  - Dapr (Event streaming)
  
Frontend:
  - React Native
  - Expo (AV, Haptics, WebSocket)
  - React Context API
  - date-fns


DATABASE SCHEMA
═══════════════

transaction_notifications
├── id (uuid)
├── transaction_id (string)
├── agent_id (string)
├── tenant_id (string)
├── amount (float)
├── transaction_type (credit/debit)
├── sender_name (string)
├── notification_sent_at (timestamp)
├── was_delivered (boolean)
└── delivered_at (timestamp)


WEBSOCKET MESSAGE TYPES
════════════════════════

Client → Server:
  - ping              (keep-alive)
  - location_update   (geofencing)
  - ack               (delivery confirmation)

Server → Client:
  - pong              (keep-alive response)
  - transaction_ping  (transaction notification)
  - geofence_violation (location alert)
