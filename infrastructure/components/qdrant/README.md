# Qdrant Vector Database

Semantic/vector search layer for AI-powered features in the 54agent platform.

## Collections

| Collection                  | Dimensions | Model                | Purpose                                      |
|-----------------------------|-----------|----------------------|----------------------------------------------|
| `agent_embeddings`          | 384       | all-MiniLM-L6-v2     | Agent behaviour profile for churn prediction |
| `customer_profiles`         | 384       | all-MiniLM-L6-v2     | Customer 360° similarity matching            |
| `compliance_documents`      | 1536      | text-embedding-3-small | Regulatory document semantic search        |
| `territory_features`        | 128       | custom               | Territory clustering ML features             |

## Quick start (dev)

```sh
cd infrastructure/components/qdrant
docker compose up -d
# API available at http://localhost:6333
# Dashboard at http://localhost:6333/dashboard
```

## Collection creation example

```sh
curl -X PUT http://localhost:6333/collections/agent_embeddings \
  -H 'Content-Type: application/json' \
  -d '{
    "vectors": {
      "size": 384,
      "distance": "Cosine"
    }
  }'
```

## Integration

Services that produce/consume vectors:
- `agent-service` — writes agent behaviour embeddings on nightly batch
- `fraud-engine` — queries `agent_embeddings` for anomaly detection
- `compliance` service — indexes regulatory docs into `compliance_documents`
- Territory analytics reads from `territory_features` for cluster assignment

REST API docs: http://localhost:6333/openapi (auto-generated by Qdrant)
